\begin{abstract}
This document is meant to help beginners and intermediates learn how to write code in C\# using the .NET framework. C\# is a modern, object-oriented programming language developed by Microsoft that makes it easy and intuitive to build robust applications. C\# is versatile and powerful, making it a great way to learn the fundamentals of programming and builds a strong foundation for learning other languages in the future. By the end of this document the reader should have a solid grasp on the syntax of the language, as well as how to work with the .NET framework and create various types of applications.
\end{abstract}

\section{Introduction}

\subsection{What is C\#?}
C\# (pronounced "C Sharp") is a modern, object-oriented programming language developed by Microsoft in 2000 as part of its .NET initiative. It was designed by Anders Hejlsberg and his team to be simple, powerful, and type-safe. C\# combines the computing power of C++ with the programming ease of Visual Basic, making it an ideal language for both beginners and experienced developers.

C\# is widely used for developing desktop applications, web applications, mobile apps, games (especially with Unity), and cloud-based services. It runs on the .NET framework, which provides a vast library of pre-built functionality that developers can leverage to build applications quickly and efficiently.

\subsection{Why Learn C\#?}
Learning C\# offers numerous advantages for aspiring programmers:

\begin{itemize}
    \item \textbf{Versatility:} C\# can be used to build virtually any type of application, from desktop software to mobile apps and web services.
    \item \textbf{Industry Demand:} C\# developers are in high demand across various industries, particularly in enterprise software development.
    \item \textbf{Strong Foundation:} Learning C\# teaches fundamental programming concepts that transfer well to other languages like Java, C++, and JavaScript.
    \item \textbf{Rich Ecosystem:} The .NET framework provides extensive libraries, tools, and community support.
    \item \textbf{Modern Features:} C\# continuously evolves with modern programming features like async/await, LINQ, pattern matching, and more.
    \item \textbf{Game Development:} C\# is the primary language for Unity, one of the most popular game development engines.
\end{itemize}

\subsection{The Foundation of C\#}
C\# is built on several key principles that make it powerful and accessible:

\subsubsection{Object-Oriented Programming}
C\# is fundamentally object-oriented, meaning it organizes code into objects that contain both data and behavior. This approach makes code more modular, reusable, and easier to maintain. The four pillars of OOP in C\# are:
\begin{itemize}
    \item \textbf{Encapsulation:} Bundling data and methods that operate on that data within a single unit (class)
    \item \textbf{Inheritance:} Creating new classes based on existing ones
    \item \textbf{Polymorphism:} The ability of objects to take multiple forms
    \item \textbf{Abstraction:} Hiding complex implementation details and showing only necessary features
\end{itemize}

\subsubsection{Type Safety}
C\# is a strongly-typed language, which means every variable must have a defined type. This helps catch errors at compile-time rather than runtime, making your programs more reliable and easier to debug.

\subsubsection{The .NET Framework}
The .NET framework is a software development platform that provides:
\begin{itemize}
    \item Common Language Runtime (CLR) for executing programs
    \item Base Class Library (BCL) with thousands of pre-built classes
    \item Tools for debugging, testing, and deployment
\end{itemize}

\section{Getting Started}

\subsection{Installing Visual Studio}
To begin programming in C\#, you'll need an Integrated Development Environment (IDE). Visual Studio is Microsoft's flagship IDE and the most popular choice for C\# development.

\subsubsection{Downloading Visual Studio}
\begin{enumerate}
    \item Visit the official Visual Studio website: \texttt{https://visualstudio.microsoft.com/}
    \item Choose Visual Studio Community (free for students and individual developers)
    \item Download the installer for your operating system
    \item Run the installer once the download is complete
\end{enumerate}

\subsubsection{Installation Process}
When you run the Visual Studio installer, you'll be presented with various workload options:

\begin{itemize}
    \item \textbf{.NET desktop development:} For building Windows desktop applications
    \item \textbf{ASP.NET and web development:} For creating web applications
    \item \textbf{Mobile development with .NET:} For building mobile apps
    \item \textbf{Game development with Unity:} For creating games
\end{itemize}

For beginners, we recommend starting with ".NET desktop development" which includes everything needed to build console and desktop applications.

\subsection{Creating Your First Project}
Once Visual Studio is installed, you're ready to create your first C\# project.

\subsubsection{Starting a New Project}
\begin{enumerate}
    \item Open Visual Studio
    \item Click on "Create a new project"
    \item In the search box, type "Console App"
    \item Select "Console App (.NET Core)" or "Console App (.NET Framework)"
    \item Click "Next"
\end{enumerate}

\subsubsection{Configuring Your Project}
After selecting the project template:
\begin{enumerate}
    \item Enter a project name (e.g., "MyFirstCSharpApp")
    \item Choose a location to save your project
    \item Optionally create a solution name
    \item Click "Create"
\end{enumerate}

Visual Studio will generate a basic project structure with a \texttt{Program.cs} file containing a simple "Hello World" program.

\subsection{Running Your First Program}
Your first C\# program is automatically generated when you create a console application. Here's what the code looks like:

\begin{verbatim}
using System;

namespace MyFirstCSharpApp
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello World!");
        }
    }
}
\end{verbatim}

\subsubsection{Understanding the Code}
Let's break down what each part means:

\begin{itemize}
    \item \texttt{using System;} - Imports the System namespace, which contains fundamental classes
    \item \texttt{namespace MyFirstCSharpApp} - Organizes your code into a logical group
    \item \texttt{class Program} - Defines a class named Program
    \item \texttt{static void Main(string[] args)} - The entry point of your application
    \item \texttt{Console.WriteLine("Hello World!");} - Prints text to the console
\end{itemize}

\subsubsection{Running the Program}
To execute your program:
\begin{enumerate}
    \item Press F5 or click the "Start" button (green arrow) in the toolbar
    \item Visual Studio will compile and run your program
    \item A console window will appear displaying "Hello World!"
    \item Press any key to close the console window
\end{enumerate}

\subsection{Understanding the Visual Studio Interface}
Visual Studio provides a powerful development environment with several key components:

\begin{itemize}
    \item \textbf{Solution Explorer:} Shows the structure of your project and all its files
    \item \textbf{Code Editor:} Where you write your code, with syntax highlighting and IntelliSense
    \item \textbf{Output Window:} Displays build messages and debugging information
    \item \textbf{Error List:} Shows compilation errors and warnings
    \item \textbf{Properties Window:} Displays and allows editing of selected item properties
    \item \textbf{Toolbar:} Quick access to common commands like Build and Run
\end{itemize}

\subsection{Common Keyboard Shortcuts}
Learning keyboard shortcuts will significantly improve your productivity:

\begin{itemize}
    \item \textbf{F5:} Start debugging
    \item \textbf{Ctrl + F5:} Start without debugging
    \item \textbf{Ctrl + S:} Save current file
    \item \textbf{Ctrl + Shift + B:} Build solution
    \item \textbf{Ctrl + K, Ctrl + C:} Comment selected lines
    \item \textbf{Ctrl + K, Ctrl + U:} Uncomment selected lines
    \item \textbf{F12:} Go to definition
    \item \textbf{Ctrl + Space:} Trigger IntelliSense
\end{itemize}

\section{Basic Syntax}
This section covers the foundational syntax elements of C\# that every beginner should know.

\subsection{Variables and Types}
C\# is a statically typed language: each variable has a type known at compile time. Common built-in types include:

\begin{itemize}
    \item \texttt{int} -- integer numbers (e.g., 0, -3, 42)
    \item \texttt{double} -- double-precision floating point numbers (e.g., 3.14)
    \item \texttt{bool} -- boolean values: \texttt{true} or \texttt{false}
    \item \texttt{string} -- sequences of characters (text)
    \item \texttt{char} -- single character
\end{itemize}

Declare and initialize variables:
\begin{verbatim}
int count = 10;
double pi = 3.1415;
bool isReady = true;
string name = "Alice";
\end{verbatim}

\subsection{Control Flow}
C\# supports the standard control flow constructs:

\begin{itemize}
    \item \textbf{Conditional:} \texttt{if}, \texttt{else if}, \texttt{else}
    \item \textbf{Switch:} \texttt{switch} for multi-way branching
    \item \textbf{Loops:} \texttt{for}, \texttt{while}, \texttt{do-while}, and \texttt{foreach}
\end{itemize}

Example:
\begin{verbatim}
for (int i = 0; i < 5; i++)
{
    if (i % 2 == 0)
        Console.WriteLine(i + " is even");
    else
        Console.WriteLine(i + " is odd");
}
\end{verbatim}

\subsection{Methods and Functions}
Methods (functions attached to classes) have a return type, a name, and optional parameters. Example:
\begin{verbatim}
static int Add(int a, int b)
{
    return a + b;
}

int result = Add(2, 3); // result == 5
\end{verbatim}

\subsection{Comments and Formatting}
Use \% for single-line comments. For longer explanations, use C-style block comments \texttt{/* ... */}.

\subsection{Common Operators}
Arithmetic: \texttt{+ - * / \%}, comparison: \texttt{== != > < >= <=}, logical: \texttt{\&\& || !}.


\section{Object-Oriented Programming}
This section expands on the object-oriented features of C\#, including how to define and use classes, inheritance, and interfaces.

\subsection{Classes and Objects}
A class is a blueprint for objects. It can contain fields, properties, methods and constructors.

\begin{verbatim}
class Person
{
    // field
    private string name;

    // property
    public int Age { get; set; }

    // constructor
    public Person(string name, int age)
    {
        this.name = name;
        Age = age;
    }

    // method
    public void Greet()
    {
        Console.WriteLine($"Hello, my name is {name} and I'm {Age}.");
    }
}
\end{verbatim}

\subsection{Encapsulation and Access Modifiers}
Use \texttt{private}, \texttt{protected}, \texttt{internal}, and \texttt{public} to control access. Prefer properties for exposing state rather than public fields.

\subsection{Inheritance and Polymorphism}
Classes can inherit from a single base class and override virtual methods to provide polymorphic behavior.

\begin{verbatim}
class Animal { public virtual void Speak() => Console.WriteLine("..."); }
class Dog : Animal { public override void Speak() => Console.WriteLine("Woof"); }

Animal a = new Dog();
a.Speak(); // prints "Woof" because of polymorphism
\end{verbatim}

\subsection{Interfaces}
Interfaces declare a contract without implementation. A class can implement multiple interfaces.

\begin{verbatim}
interface ILogger { void Log(string message); }
class ConsoleLogger : ILogger { public void Log(string message) => Console.WriteLine(message); }
\end{verbatim}

\subsection{Best Practices}
Keep classes small and focused, prefer composition over inheritance when appropriate, and use interfaces to express contracts for testability.
